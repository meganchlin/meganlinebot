1. Introduction to the problem

Motivation:

近兩年來，全球最關心的議題莫過於「COVID-19」，尤其台灣近期更飽受 新冠肺炎之苦，疫情的肆虐導致人們的生活方式被迫改變，每天最關注的話題無 非是新增的確診案例及死亡人數；同時大家也期望自己不會成為下一個確診者， 所以對於「是否確診的醫療檢測準確度及效率」也非常關注，因此我們以這個為 出發點，建構一個以肺部照片預測是否確診的模型，希望能提升醫療檢測的效率， 也期望台灣能早日渡過這波疫情的難關。

Introduction:

利用 CNN 做 image classification，對一系列「可能患者的肺部電腦斷層 掃描圖片（lung CT scan）」進行分類（圖片資料集來源為 Kaggle），判斷並 預測他是否有罹患 COVID-19，以增加醫療檢測的效能。

3. Algorithm

Convolutional Neural Network (CNN) :

(1)下載資料集：

從 Kaggle 下載 dataset，並將檔案解壓縮至「train」資料夾。

（網址：https://www.kaggle.com/maedemaftouni/large-covid19-ct-slice-

dataset）

(2) 處理資料：

Train: 將資料以 4:1 的比例分為「訓練(training)」及「驗證(validation)」 兩個部分，每張圖片的大小(image_size)為 512x512 像素，並設定批量大 小(batch_size)為一次讀取 32 張圖片。

Test: 從資料集中隨機取 1000 筆資料，並將這些資料移至「test」資料夾 中（即用於訓練的資料集中不會包含這 1000 筆資料）；再將圖片中每像素 的數值正規化(rescale)，即所有資料乘上 1.0/255。

(3)建構卷積神經網路模型：

• 第一卷積層：產生 32 組 3x3x3 濾波器並搭配 ReLu 激活(Activation) 函數，每一組濾波器再加上一個 bias 一起訓練，第一次共有(3x3x3 +

1)x 32 = 896 個參數待訓練，卷積後圖像大小變為 512x512x32，第 二次共有(3x3x32 + 1) x 32 = 9248 個參數，圖像大小不變；接著以 2x2 窗 口 進 行 最 大 池 化 (Max Pooling) ， 並 將 圖 像 縮 小 至 一 半 尺 寸 256x256x32。

•第二卷積層：產生 64 組 3x3x3 濾波器並搭配 ReLu 激活函數，每一組 濾波器再加上一個 bias 一起訓練，第一次共有(3x3x32 + 1) x 64 = 18496 個參數，卷積後圖像大小變為 256x256x64，第二次共有(3x3x64 + 1) x 64 = 36928 個參數，圖像大小不變；再以 2x2 窗口進行最大池 化，並將圖像縮小至 128x128x64，池化後使用 Dropout 正則化以避 免 overfit，dropout rate = 0.25。

•第三卷積層：產生 128 組 3x3x3 濾波器並搭配 ReLu 激活函數，每一 組濾波器再加上一個 bias 一起訓練，第一次共有(3x3x64 + 1) x 128 =

73856 個參數，卷積後圖像大小變為 126x126x128，第二次共有 (3x3x128 + 1) x 128 = 147584 個參數，圖像大小變為 124x124x128；

再以 2x2 窗口進行最大池化，並將圖像縮小至 62x62x128，池化後使 用 Dropout 正則化以避免 overfit，dropout rate = 0.35。

•將所有節點展開(Flatten)變成一維節點，共有 62x62x128 = 492032 個 節點。

•以 256 個節點和上一層進行 全 連 結 (Dense) ， 並 搭 配

ReLu 激活函數，加上一個 bias 一起訓練，共有(492032 + 1) x 256 = 125960448 個 參數。

•最後共有 126247970 個參數 待訓練。

(4)Compile the model：

配置模型訓練及優化的規格。

(5)訓練及驗證 model：

利用先前準備好的資料集，對 model 進行訓練與驗證。設定迭代次數 (epochs)為 10，在執行過程中，每次迭代會輸出訓練的損失率(loss)、正確 率(acc)及驗證的損失率(val_loss)及正確率(val_acc)。

(6)繪製折線圖：

利用訓練過程收集到的數據以折線圖方式繪製出兩個圖表，以方便了解訓練 及驗證過程的正確率(Accuracy)及損失率(Loss)。第一個折線圖為正確率， 第二個為損失率，橫軸代表迭代次數(Epoch)，縱軸代表正確(或損失)率，而 藍線代表訓練集結果，橘線代表驗證集結果。

(7)預測 test 的資料：

利用訓練好的 model 對 test 的資料圖集進行預測，並將預測結果記錄到 predictions list 中。每次選一張圖片，假如預測為 COVID-19 確診者便存 入「COVID」，預測不是確診者則存入「NonCOVID」。
